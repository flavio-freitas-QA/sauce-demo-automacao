# Dia 001 — Fluxo de Login (Sauce Demo)

**Ferramentas:** Cypress + Playwright (comparativo)
**Aplicação alvo:** [saucedemo.com](https://www.saucedemo.com)

## Contexto

Primeiro dia do portfólio. Objetivo foi estabelecer o padrão de estrutura (Page Object Model, fixtures, comandos customizados) que será reutilizado nos próximos dias, cobrindo o fluxo de login em duas ferramentas diferentes para efeito de comparação.

## Cenários cobertos

- Login com sucesso (usuário `standard_user`)
- Login com senha inválida
- Login com usuário bloqueado (`locked_out_user`)
- Validação de campos obrigatórios
- Logout após login bem-sucedido (apenas Cypress)

## Decisões técnicas

- **Page Object Model** desde o primeiro dia, para evitar refatoração grande mais adiante.
- Fixture `users.json` centraliza as credenciais de teste, evitando strings soltas nos specs — Sauce Demo já disponibiliza usuários com propósitos específicos (`locked_out_user`, `problem_user`, etc.), o que facilita cobrir cenários negativos sem precisar de mocks.
- Cypress: usei um comando customizado (`cy.login`) além do Page Object, para servir de referência de quando optar por um ou outro (comando é mais direto para ações repetidas simples; POM escala melhor para telas complexas).

## Comparativo Cypress vs Playwright (primeiras impressões)

| Aspecto | Cypress | Playwright |
|---|---|---|
| Sintaxe | Encadeada (`cy.get().type()`) | `async/await` explícito |
| Fixtures | `cy.fixture()` + alias | Import direto de JSON |
| Múltiplos browsers no mesmo run | Não nativo | Nativo (`projects`) |
